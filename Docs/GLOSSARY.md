## Terms

- **BYOK (bring-your-own-key)**: Hark runs no servers of its own: the user supplies their own speech-to-text provider key (OpenAI, Groq, or Deepgram), and, for non-Verbatim voices, their own LLM key for cleanup. Both are entered in Settings on first run and stored in the OS keychain, never in `config.toml` ([README.md:3](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L3), [CLAUDE.md:3](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L3)).
- **Cleanup pass**: The optional, single low-temperature LLM call that rewrites the raw transcript according to the selected voice template plus dictionary terms; skipped entirely when the voice is Verbatim ([README.md:27,50](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L27), [CLAUDE.md:17](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L17)).
- **Clipboard injection**: The default text-insertion strategy: stash the clipboard, set it to the polished transcript, paste, then restore the original clipboard contents, so the cursor's app receives normal keystroke-level paste input ([README.md:28,52](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L28), [CLAUDE.md:18](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L18)).
- **Deepgram adapter**: The `SttProvider` implementation for Deepgram's nova-3 model, using `keyterm` biasing for dictionary terms; it is a distinct code path from the OpenAI-compatible adapter because Deepgram's API shape differs ([README.md:24](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L24), [CLAUDE.md:14](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L14)).
- **Dictionary**: The user's local list of terms (names, jargon) applied two ways: phonetic post-correction after transcription (provider-agnostic) and per-provider biasing sent with the request (`prompt` for OpenAI/Groq, `keyterm` for Deepgram) ([README.md:26](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L26), [CLAUDE.md:16](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L16)).
- **enigo**: The keystroke-simulation crate used as Hark's injection fallback when clipboard paste is unavailable or fails ([README.md:28](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L28), [CLAUDE.md:18](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L18)).
- **keyterm**: Deepgram's unweighted keyword-boosting mechanism, available on nova-3 and later models, used to bias transcription toward dictionary terms ([README.md:24,26](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L24), [CLAUDE.md:14,35](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L14)).
- **keywords**: Deepgram's legacy *weighted* keyword-boosting mechanism, tied to the nova-2 model generation; mutually exclusive with `keyterm` because the two belong to different model generations ([CLAUDE.md:35](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L35)).
- **Launch at login**: The optional Windows-registry (or macOS login item) registration that starts Hark hidden in the system tray when the user signs in, toggled under Settings → Behavior ([README.md:73,114,135](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L73)).
- **Main-thread rule**: The project's one hard threading constraint: the UI event loop (tray + egui) owns the main thread, while the entire dictation pipeline (hotkey, audio, STT, HTTP, injection) runs on worker threads only, because macOS requires all UI on the main thread ([CLAUDE.md:11,26](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L11), [README.md:57](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L57)).
- **OpenAI-compatible adapter**: The `SttProvider` implementation for the shared multipart `/audio/transcriptions` contract used by both OpenAI and Groq, distinguishing it from the separately-shaped Deepgram adapter ([README.md:24](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L24), [CLAUDE.md:14](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L14)).
- **Phonetic post-correction**: The primary, provider-agnostic dictionary mechanism: after transcription returns, Hark matches words phonetically against the dictionary to fix misheard names and jargon, independent of which STT provider was used ([README.md:26](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L26), [CLAUDE.md:16](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L16)).
- **Pre-roll**: The roughly 200-300 ms of audio captured into the ring buffer *before* the push-to-talk key is registered as down, so the first syllable of speech is never clipped ([README.md:22,38](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L22), [CLAUDE.md:12](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L12)).
- **prompt biasing**: The OpenAI/Groq mechanism for steering transcription toward dictionary terms, passed via the `prompt` parameter on the multipart request (the OpenAI-compatible counterpart to Deepgram's `keyterm`) ([README.md:26](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L26), [CLAUDE.md:16](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L16)).
- **Push-to-talk (PTT)**: Hark's core interaction model: hold a key, speak, release the key, and polished text is injected at the cursor; implemented with native low-level key hooks (`CGEventTap` on macOS, `WH_KEYBOARD_LL` on Windows) rather than the `global-hotkey` crate, for reliable held-key press/release edges ([README.md:1,3,23](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L1), [CLAUDE.md:3,13](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L3)).
- **Recording overlay**: A small, always-on-top, mouse-transparent "pill" window that appears near the bottom of the screen while the push-to-talk chord is held, with a circle that pulses to live mic input; implemented as an egui deferred viewport so it stays on the main thread like the rest of the UI ([overlay.rs:1-8](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/crates/hark-app/src/overlay.rs#L1-L8)).
- **Release-to-inject**: The latency window Hark optimizes for: everything that happens between the push-to-talk key being released and the polished text appearing at the cursor (WAV encode, one HTTPS POST to the STT provider, optional cleanup call, inject) ([README.md:9,44-52](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L9), [CLAUDE.md:27](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L27)).
- **Ring buffer**: The `cpal`-backed, fixed-size 16 kHz mono audio buffer that continuously holds recent microphone samples, enabling pre-roll capture without a dedicated "start recording" delay ([README.md:22,38](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L22), [CLAUDE.md:12](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L12)).
- **Self-update**: The in-app update checker's Windows-specific capability to download and apply a new release without the user visiting the Releases page manually ([README.md:115](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L115)).
- **Silence trimming**: The step, after the tail is appended and before WAV encoding, that trims leading and trailing silence from the captured utterance so the STT request carries only the spoken audio ([README.md:42](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L42)).
- **Single-instance guard**: The Phase 5 mechanism ensuring only one Hark process runs at a time, preventing duplicate tray icons or competing hotkey hooks ([README.md:102,135](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L102)).
- **SttProvider**: The Rust trait abstracting speech-to-text providers behind a common interface, with the OpenAI-compatible and Deepgram adapters as its two Phase 1 implementations ([README.md:24](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L24), [CLAUDE.md:14](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L14)).
- **Tail**: The roughly 150 ms of audio appended after the push-to-talk key is released, capturing any trailing syllable before the ring buffer stops recording ([README.md:22,40](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L22), [CLAUDE.md:12](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L12)).
- **Tray daemon**: The always-on background process that owns Hark's hot path (hotkey, audio, STT, injection); the settings/history window is a separate on-demand view onto the same main-thread event loop ([README.md:57](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L57)).
- **Verbatim**: The voice option that skips the cleanup pass entirely and injects the raw, phonetically-corrected transcript unchanged; it is the only voice implemented in Phase 1 ([README.md:47,131](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L47)).
- **Voice / Voice preset**: A named cleanup template (Verbatim being the no-op case) that combines a prompt style with dictionary terms for the BYOK LLM cleanup call; voices are data, not code, so editing them never touches the pipeline ([README.md:27,50,133](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L27), [CLAUDE.md:17](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L17)).
- **Worker thread**: Any thread other than the main UI thread; the entire dictation pipeline (hotkey capture, audio, STT HTTP calls, injection) runs here so it never blocks the tray/egui event loop ([CLAUDE.md:11,26](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/CLAUDE.md#L11), [README.md:21,57](https://github.com/BoardPandas/Hark/blob/1c1738716fa4cd758b0c26ec94d0873d1bc35ac1/README.md#L21)).
